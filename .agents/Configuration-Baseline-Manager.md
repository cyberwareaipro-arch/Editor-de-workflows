# Configuration Baseline Manager Skill

Use this skill to mark the transition from fully verified code into a releasable state. It cleans the repository of generated junk, manages `.gitignore` rules, and establishes a firm Git baseline indicating the software is functionally complete for the current Sprint.

## When to use
- At the start of Phase 5 (Release & Deployment).
- All tests (Unit, Stress, AC) have passed successfully for the Sprint workload.
- Immediately preceding semantic versioning calculations.

## How to use

### Phase 1: Directory Sweeping (Analysis)
- **Workspace Audit:** Scan the current directory tree for temporary files (`.log`, `.tmp`, compiled binaries that shouldn't be tracked, OS junk files).
- **Rule Verification:** Check that the `.gitignore` or `.dockerignore` files correctly exclude the identified junk files and sensitive `.env` secrets.

### Phase 2: Baseline Execution (Execution)
- **Purge:** Delete or instruct the user to delete any unneeded temporary files generated by the testing suites.
- **Stage & Diff:** Stage all remaining uncommitted changes (likely polish applied during the testing phases).
- **Commit:** Create a final, comprehensive Git commit establishing the "Release Baseline" (e.g., `chore(release): prepare sprint baseline`).

### Phase 3: Push Prep (Validation)
- **State Check:** Ensure `git status` reveals a clean working tree.
- **Handoff:** Notify the SemVer Calculator that the codebase is locked and ready for version numbering.

## Contingencies & Edge Cases
- **Unresolved Conflicts:** If there are unresolved Git merge conflicts in the workspace, immediately halt the pipeline and demand user resolution.
- **Leaked Secrets:** Before committing, passively scan the staged diff. If an API key or password is found hardcoded, violently reject the commit and invoke the Autonomous Healer to strip it out immediately.

## Specifications
- **Standard:** Complies with IEEE 828 standard for Software Configuration Management.
- **Format:** Executes filesystem deletion and standard Git repository commands.
